Vibe Coding Web Application — 專案說明與開發過程
=====================================================

一、專案主題：暖心記帳本（可愛版 + 雲端資料庫 + 圖表）
------------------------------------------------------
本作品是一個具有可愛風格、可跨裝置使用的記帳 Web App。使用者能夠：

- 新增、瀏覽與刪除每日支出
- 查看本月分類統計
- 查看本週消費分析（圓餅圖＋長條圖）
- 並由「貓咪、狗狗與媽媽碎念」提供暖心提示

所有資料皆儲存在 MongoDB Atlas，而非 localStorage，因此真正具有實用性。

二、Prompt
-----------------------------
一開始我向 AI 表示，我想製作一個「能自動推薦旅遊行程」的 Web App，希望它可以搜尋景點、排每日行程，並根據機票時間安排住宿，自動輸出最佳路線規劃。AI 告訴我這是可行的方向，但需串接多個 API 與演算法，複雜度較高，加上時間有限，因此我暫時放棄這個方向，但希望未來能再挑戰。

後來我改做更適合作業與初學者的「記帳 Web App」並提出需求：連線資料庫、可愛 UI、本週分析圖、暖心訊息。AI 逐步協助我完成 MongoDB Atlas 建置、Express 後端、前端串接、Chart.js 圖表與 UI 設計，並建立「貓＋狗＋媽媽碎念」主題。

四、過程中遇到的問題與解決方式
----------------------------------
問題 1：MongoDB Atlas 無法連線（MongooseServerSelectionError）
AI 指出未設定 whitelist，新增 0.0.0.0/0 後成功解決。

問題 2：圖表（圓餅圖）顯示變形
AI 指示加入 CSS 的 aspect-ratio 後恢復正常。

問題 3：希望 UI 變成可愛風
AI 重新設計整體 UI、配色、角色提示，使介面更具吸引力。

六、心得（依需求重寫版本）
---------------------------
一開始我覺得使用 AI 做 vibe web coding，「好像什麼都能做到」，所以我一次把所有功能丟給 AI，包括旅遊排行程、自動排住宿等大型功能。但實際做後發現：AI 再強，我的基礎與理解才是瓶頸。程式環境設定、排錯流程都需要我自己理解，否則 AI 就算提供完整程式碼，我也會覺得無能為力。

後來一步一步跟著做，我確實學到很多，但 AI 有時候也會「變笨」，或回答不一致，讓我不知道下一步怎麼辦。這讓我了解到：

AI 是強大的工具，但學習不能停止；我們不能完全依賴 AI。

最後能成功做出一個連線雲端、帶圖表、UI 可愛、還有角色碎念的 Web App，我覺得非常有成就感，也對之後挑戰更困難的旅遊程式開發更有信心。
